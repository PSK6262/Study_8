Q1.서버와 클라이언트란 무엇인가?
Q2.서버와 클라이언트의 관계는 어떻게 되는가?

A.
클라이언트와 서버는 서로 요청(request)과 응답(response)을 주고받는 관계
클라이언트는 서버에 요청을 보내고, 서버는 그 요청을 처리하여 결과를 응답한다
클라이언트는 사용자와 상호작용하며 데이터를 화면에 표시하는게 주된 역할이고
서버는 DB를 이용한 데이터 저장과 연산을 담당한다
주된 계산들을 전부 서버가 맡는다는 오래된 인식이 있지만
JavaScript의 계산은 클라이언트쪽에서 많이 하는걸 생각하면 그렇지만도 않다
이런 구조는 효율적인 자원관리(분할업무)와 보안성을 위해 고안된 것이다

Q3.Live Server 로 실행시킬때 나오는 이 주소의 의미가 서버라는 개념과 어떤 관계가 있는가?
http://127.0.0.1:5500/fe_study/html/start.html
Q4.Live Server 확장프로그램을 설치해서 사용하고 있는데,
Live Server는 Server 라는 이름을 가지고 있다. Live Server 가 제공해주는 기능은 무엇인가?

A.
HTTP는 클라이언트와 서버가 통신하기 위해서 만들어 둔 규칙 같은 것
127.0.0.1은 자기 자신의 IP이며 Loopback IP, localhost등 여러가지로 불린다
5500은 포트번호로, 요청을 받을 서버 프로그램을 구분하기 위한 것
뒤에 나오는 fe_study/html/start.html는 클라이언트가 html 파일을 요청하는 과정
이후에 서버에서 내용을 전달 해 주면 클라이언트가 화면에 표시하게 된다

Live Server는 내 컴퓨터 내에서 서버를 띄워주는 역할을 한다
내 컴퓨터 자체가 서버가 되는 것이 아니고, 내 컴퓨터에서 실행한 Live Server가 서버 역할을 해 주는 것이다
비슷하게 보일 수 있지만 엄연히 다른 것이므로 혼동하지 말아야 한다.
